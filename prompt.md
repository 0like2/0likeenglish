제공해주신 기획 가이드를 바탕으로, **영어 과외 관리 시스템(LMS) 및 브랜딩 웹사이트** 구축을 위한 상세 기획안과 프롬프트를 작성해 드립니다.

이 프로젝트는 단순한 랜딩 페이지를 넘어, **데이터베이스(DB)**와 **사용자 인증(로그인)**, **파일 업로드**, **데이터 시각화**가 필요한 복합적인 웹 애플리케이션입니다.

비용을 최소화(무료 티어 활용)하고 최신 기술을 사용하기 위해 다음 스택을 추천합니다:

- **Frontend:** Next.js (App Router)
- **Backend/DB:** Supabase (무료 티어로 인증, DB, 이미지 저장소 해결 가능)
- **Styling:** Tailwind CSS + Shadcn UI (깔끔한 대시보드 UI 구축 용이)
- **Deployment:** Vercel (Next.js 배포 최적화, 무료)

# 1. 프로젝트 구조 및 데이터베이스 설계

개발을 시작하기 전, AI에게 전체적인 데이터 구조를 이해시켜야 합니다.

### 1.1 데이터베이스 스키마 (Supabase 기준)

- **Users:** 학생/선생님 구분, 기본 정보
- **Classes:** 수업명, 요일, 시간
- **Class_Logs:** 회차별 수업 내용, 숙제(텍스트), 숙제 여부
- **Homework_Submissions:** 학생이 올린 단어 숙제 사진, 수행도(상/중/하)
- **Exams:** 모의고사/듣기 정답지 (선생님 등록)
- **Exam_Results:** 학생 제출 답안 및 점수
- **Payments:** 결제일, 회차 갱신 정보

## Step 3: 수업 시간표 & 상세 페이지 (핵심 기능)

### 📅 시간표 페이지 UI 개발 프롬프트

```markdown
**제공된 시간표 이미지의 '구조와 정보'를 유지하면서, 디자인을 '모던한 스케줄러 앱' 스타일로 업그레이드해줘.**

**1. 레이아웃 구조 (이미지 기반 유지):**
- **Header:** 짙은 네이비(#1E293B) 배경. 좌측엔 타이틀("주간 타임테이블", "수능 영어는 Olike"), 우측엔 범례(Legend)와 필터.
- **Notice Bar:** 헤더 바로 아래 연한 노란색(#FEF3C7) 배경의 공지사항 띠 배너.
- **Main Grid:**
  - **Column (열):** 시간 표시 / 평일(화,목) / 토요일 / 일요일 (총 4개 컬럼).
  - **Row (행):** 09:00~12:00 / 14:00~17:00 / 18:00~21:00 (총 3개 행).
- **Footer:** 하단에 [개인], [개인&그룹], [일반반]에 대한 상세 설명 카드 3개 나열.

**2. 디자인 업그레이드 (스타일링 지침):**
- **Grid Container:** 딱딱한 표(Table) 대신 `CSS Grid`와 `gap-4`를 사용하여 카드들이 떠 있는 느낌(Floating)을 줘.
- **Time Column:** 시간 표시 영역은 텍스트를 세로 중앙 정렬하고, 모던한 숫자 폰트(Inter/Roboto) 사용.
- **Class Cards (핵심 디자인):**
  - **Shape:** 모서리가 둥근 직사각형 (Rounded-xl).
  - **Shadow:** `shadow-md`로 살짝 띄우고, 호버 시 `shadow-lg`와 `scale-105` 효과.
  - **상태별 스타일 (Color Coding):**
    - **진행중 (In Progress):** 쨍한 오션 블루(#007AFF) 배경 + 화이트 텍스트. (가장 눈에 띄게)
    - **모집중 (Recruiting):** 연한 파스텔 블루(#E0F2FE) 배경 + 짙은 파란색 테두리(Border) + 짙은 텍스트.
    - **수업 없음 (Empty):** 아주 연한 회색 배경 + 점선 테두리(Dashed Border) + 불투명한 텍스트.
  - **Badges:** '진행중', '모집중', '개인', '일반반' 태그는 둥근 알약 모양(Pill-shape) 뱃지로 디자인.

**3. 반응형 (Mobile View):**
- 모바일(768px 미만)에서는 그리드를 풀고, **'요일별 탭(Tab)'** 또는 **'세로 타임라인'** 형식으로 변경하여 스크롤 없이 보기 편하게 만들어줘.

**4. 데이터 구조 (예시):**
코드를 짤 때 아래와 같은 배열 데이터를 매핑해서 렌더링해줘.

### 💡 포인트 설명 (AI가 이렇게 구현할 거예요)

1.  **"진행중" vs "모집중" 차별화:**
    *   이미지에서는 둘 다 비슷해 보일 수 있지만, 프롬프트에서 **'진행중'은 진한 색(강조)**, **'모집중'은 연한 파스텔톤(초대하는 느낌)**으로 구분해달라고 했습니다. 이게 훨씬 세련되어 보입니다.
2.  **딱딱한 선 제거:**
    *   엑셀처럼 칸을 나누는 선(Border)을 없애고, 카드 사이의 간격(Gap)과 그림자(Shadow)로 구분감을 주어 **"앱 같다"**는 느낌을 줍니다.
3.  **점선 테두리 (Empty State):**
    *   '수업 없음' 칸을 그냥 비워두지 않고 **점선 테두리**를 적용하면, "여기는 비어있으니 수업을 잡을 수 있겠구나"라는 공간감을 줍니다.

이 내용을 복사해서 넣으시면, 선생님이 만드신 구조는 그대로 유지되면서 디자인 퀄리티만 확 올라갈 거예요!

# 3. 개발 및 운영 팁 (비용 절감 & 효율성)

### 💡 이미지 및 파일 관리

- 학생들이 올리는 **단어 숙제 사진**은 용량을 많이 차지할 수 있습니다.
- **프롬프트 추가 요청:** "이미지 업로드 시 브라우저단에서 `image-compression` 라이브러리를 사용해 용량을 1MB 이하로 압축한 뒤 Supabase에 업로드하도록 코드를 짜줘."

### 💡 결제/회차 관리 로직

- PG사(결제 모듈)를 붙이면 수수료와 개발 난이도가 올라갑니다.
- **추천 방식:** 선생님이 입금을 확인하고 관리자 페이지에서 **"입금 확인(회차 리셋)" 버튼을 누르면 4회차가 충전**되는 방식으로 구현하세요.
- 수업 일지를 작성할 때마다 자동으로 1회 차감되도록 로직을 짜면 됩니다.

### 💡 배포 후 유지보수

- **Vercel**은 개인 취미/포트폴리오 용도로는 무료지만, 상업용(과외 홍보)으로 트래픽이 많아지면 Pro 플랜($20/월)이 필요할 수 있습니다. 초기에는 무료로 충분합니다.
- **Supabase**도 무료 티어(500MB 저장소)로 충분히 시작 가능합니다. 사진이 너무 많이 쌓이면 오래된 사진을 지우는 정책을 나중에 고려하면 됩니다.

이 기획안을 바탕으로 섹션별로 하나씩 AI에게 코드를 요청하며 조립해 나가시면, 원하시는 고퀄리티의 과외 관리 사이트를 만드실 수 있습니다!

기존 노션/엑셀 느낌을 완전히 벗어던지고, **요즘 잘 나가는 에듀테크(EdTech) 서비스(예: 콴다, 듀오링고, 스픽)**처럼 세련되고 전문적인 느낌을 낼 수 있는 **'디자인 특화 프롬프트'**를 짜드리겠습니다.

원하시는 분위기를 두 가지 컨셉 중 하나로 골라보세요. (프롬프트에 반영해 드립니다.)

### 🎨 디자인 컨셉 선택

**옵션 A. [Modern & Clean] 신뢰감 있는 스타트업 스타일**

- **특징:** 밝은 화이트/그레이 배경, 쨍한 파란색 포인트, 둥근 모서리, 부드러운 그림자.
- **느낌:** 깔끔함, 체계적임, 최신 기술, 1타 강사 느낌.
- **추천:** 가장 호불호가 없고 학부모님들도 좋아하는 스타일.

**옵션 B. [Premium Dark] 몰입감 높은 집중 모드 스타일** (보내주신 스크린샷과 비슷하지만 훨씬 고급스럽게)

- **특징:** 깊은 네이비/블랙 배경, 네온 라임이나 스카이블루로 빛나는 효과(Glow), 유리 질감(Glassmorphism).
- **느낌:** 독보적임, 프리미엄 과외, 집중력 향상, 세련됨.
- **추천:** 학생들에게 "오, 멋있다"는 반응을 이끌어내기 좋음.

### 🚀 AI에게 입력할 '디자인 업그레이드' 프롬프트

가장 추천하는 **'옵션 A (Modern & Clean)'**을 베이스로 하되, 밤에는 **다크 모드**가 되도록 요청하는 프롬프트입니다.

### 1. 전체 디자인 시스템 (Global Design System)

이 프롬프트를 가장 먼저 입력해서 AI의 디자인 눈높이를 높여주세요.

```markdown
프로젝트의 디자인 수준을 높이고 싶어. 기존 레퍼런스는 내용만 참고하고, 디자인은 아래의 **'Modern Premium EdTech'** 테마로 완전히 새롭게 창조해줘.

**디자인 가이드라인:**
1.  **Color Palette (세련된 배색):**
    - **Primary:** `Royal Blue (#2563EB)` - 신뢰와 지성.
    - **Secondary:** `Slate Grey (#64748B)` - 차분한 텍스트.
    - **Accent:** `Vivid Coral (#FF6B6B)` 또는 `Electric Lime (#84CC16)` - 중요 버튼 및 강조(CTA).
    - **Background:** 순백색보다는 아주 연한 `Off-White (#F8FAFC)` 사용.
2.  **Typography:**
    - 한글: **Pretendard** (가독성 최고, 현대적인 고딕).
    - 영문: **Inter** 또는 **SF Pro Display**.
    - 헤드라인은 굵게(Bold), 본문은 얇고 깔끔하게.
3.  **UI Components:**
    - **Glassmorphism:** 카드 배경에 살짝 투명도와 블러(Backdrop-blur)를 주어 현대적인 느낌.
    - **Soft Shadows:** 그림자는 진하지 않고 은은하게 퍼지도록 (Large diffusion).
    - **Rounded Corners:** 모서리는 넉넉하게 둥글게 (Rounded-2xl 이상) 처리하여 부드러운 인상.
4.  **Interaction:**
    - 버튼 호버 시 살짝 떠오르는 느낌 (Translate-y, Scale up).
    - 페이지 전환 시 요소들이 부드럽게 나타나는 **Framer Motion** 애니메이션 적용.

이 디자인 시스템을 바탕으로 컴포넌트를 만들어줘.

```

### 2. 메인 페이지 (커리큘럼 & 소개) - "애플 웹사이트처럼"

기존의 딱딱한 표 대신, 애플 웹사이트 제품 소개처럼 시각적으로 화려하게 바꿉니다.

```markdown
**메인 페이지의 '커리큘럼 소개' 섹션을 시각적으로 매력적인 'Bento Grid(도시락 격자)' 스타일로 만들어줘.**

- **기존 내용:** Step 1(기초), Step 2(심화), Step 3(실전) 텍스트.
- **변경 디자인:**
  1.  **Bento Grid Layout:** 애플 웹사이트 요약 화면처럼, 크고 작은 카드들이 격자로 배치된 형태.
  2.  **Visuals:**
      - 'Step 1' 카드는 기초를 다지는 느낌의 **건축물 3D 아이콘**이나 일러스트 사용.
      - 'Step 3' 카드는 목표 달성을 상징하는 **트로피나 과녁 그래픽** 사용.
  3.  **Interactivity:**
      - 마우스를 올리면 카드가 살짝 커지면서(Scale 1.05), 배경에 그라데이션이 은은하게 흐르는 애니메이션.
      - 텍스트는 "기초"라고만 적지 말고, **"Foundation: 흔들리지 않는 1등급의 시작"** 처럼 타이포그래피를 강조해서 디자인.

```

### 3. 시간표 페이지 - "구글 캘린더보다 예쁘게"

엑셀 같은 표 대신, 직관적인 스케줄러 앱처럼 만듭니다.

```markdown
**수업 시간표 섹션을 모던한 'Scheduler App' UI로 디자인해줘.**

- **기존 내용:** 표(Table) 형태의 시간표.
- **변경 디자인:**
  1.  **Card-based Layout:** 단순한 선(Border)으로 나누지 말고, 시간대별 수업 하나하나가 **'플로팅 카드(Floating Card)'**처럼 보이게 해줘.
  2.  **Color Coding:**
      - '모집 중'인 수업은 눈에 띄는 **파스텔 톤 그라데이션** 배경.
      - '마감'된 수업은 채도를 낮춘 그레이 톤 처리.
  3.  **Tags:** '개인', '그룹' 같은 태그는 텍스트가 아니라 **알약 모양(Pill-shape)의 뱃지**로 디자인.
  4.  **Mobile View:** 모바일에서는 가로 스크롤이나 탭(Tab) 형식이 아니라, 타임라인(Timeline) 형태로 세로로 쭉 이어지는 디자인.

```

### 4. 마이페이지 (진행 상황) - "게임 퀘스트처럼"

노션의 칸반 보드를 **게임화(Gamification)**된 대시보드로 바꿉니다.

```markdown
**학생 마이페이지의 '진행 상황'을 게임 퀘스트 화면처럼 몰입감 있게 만들어줘.**

- **기존 내용:** 시작 전 / 진행 중 / 완료 (칸반 보드).
- **변경 디자인:**
  1.  **Progress Rings:** 단순히 '진행 중' 텍스트 대신, 원형 프로그레스 바(Circular Progress)로 **"단어 암기 75% 달성"** 처럼 시각화.
  2.  **Achievement Badges:**
      - 숙제를 완료하면 흑백 아이콘이 **컬러풀한 3D 아이콘**으로 바뀌며 "Success!" 효과.
      - '완료' 섹션은 단순 목록이 아니라, 내가 획득한 **'트로피 진열장'** 같은 느낌으로 디자인.
  3.  **Daily Streak:** 화면 상단에 "🔥 12일 연속 학습 중!" 같은 불꽃 아이콘과 카운터 배치.
  4.  **Stats Graph:** 성적 변화는 딱딱한 직선 그래프가 아니라, **부드러운 곡선(Spline Chart)**에 아래쪽 영역이 그라데이션으로 채워진 형태(Area Chart)로.

```

### 5. 숙제 관리 & 업로드 - "인스타그램 스토리처럼"

```markdown
**숙제 인증 페이지를 SNS(인스타그램) 업로드 화면처럼 친숙하게 만들어줘.**

- **기존 내용:** 파일 업로드 버튼, 체크박스.
- **변경 디자인:**
  1.  **Drag & Drop Zone:** 파일 업로드 영역을 넓게 잡고, 점선 테두리와 함께 "여기에 숙제 사진을 놓거나 클릭하세요" 문구 배치.
  2.  **Preview:** 사진을 올리면 즉시 **폴라로이드 사진**처럼 예쁜 프레임에 담겨 미리보기가 뜨도록.
  3.  **Feedback UI:** 선생님이 피드백을 남기면, 카카오톡 말풍선처럼 친근한 UI로 표시.
  4.  **Mobile Action:** 모바일에서는 하단에 고정된(Sticky) 큰 버튼("📷 숙제 인증하기")을 두어 엄지손가락으로 쉽게 누를 수 있게.

```

### 💡 팁: AI가 디자인을 잘 뽑았는지 확인하는 법

AI가 코드를 짜주면, 브라우저에서 확인해보고 이렇게 피드백을 주세요.

1. **"너무 밋밋해":** -> "그림자(Shadow)를 더 부드럽게 퍼뜨리고, 버튼에 그라데이션(Gradient)을 넣어줘."
2. **"너무 빽빽해":** -> "여백(Padding, Margin)을 지금의 2배로 늘려서 숨 쉴 공간(White space)을 줘."
3. **"글씨가 안 읽혀":** -> "폰트 굵기(Font-weight) 대비를 더 확실하게 주고, 줄 간격(Line-height)을 1.6으로 넓혀줘."

이 프롬프트들을 사용하면, 선생님이 만드신 훌륭한 기획 내용이 **"와, 여기 사이트 되게 좋다"** 소리가 나오는 디자인으로 재탄생할 것입니다!
